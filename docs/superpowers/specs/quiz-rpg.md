# 数学冒险游戏需求总览

这份文档是主文档，只负责说明产品总方向、模块边界和子文档目录。详细需求拆到子文档中，后续讨论和实现优先直接修改对应模块文档，而不是继续把所有内容堆回一份总稿。

## 当前产品方向

当前方向已经明确收敛为：

- `主线核心`：常驻世界自由探索的数学 RPG
- `长期成长`：主城成长、工具研究和知识树推进
- `学习载体`：战斗、机关、路径选择和资源回收都通过数学能力驱动

一句话定义：

> 玩家在数学大陆中探索、解谜、遇怪答题、带回资源建设主城，并沿着数学知识树逐步解锁更深区域与更高难度挑战。

## 项目边界

本项目只做 `数学`，不以扩展其他学科为目标。

当前 `P0` 首版世界范围明确为：

- `主城`
- `数学大陆`

后续内容增长优先来自：

- 更深的数学知识树
- 更多数学知识点区域
- 更丰富的战斗、机关和 Boss 机制

而不是来自新增学科。

## 设计总原则

首版必须同时满足 `游戏性` 和 `学习性`，并明确采用“先冒险、后出题、再反馈”的结构。

- `知识树驱动世界`：区域开放顺序由数学知识点前置关系决定，不靠纯等级推进。
- `数学能力即世界动作`：数量判断、规律识别、空间理解、关系推演直接对应战斗、解谜和路径推进。
- `先给语境，再给问题`：玩家先看到怪物、机关、门锁、桥梁、线索和路线，再进入答题环节。
- `多目标并行`：任何一次出城都不只服务主线，还应同时承载资源回收、错题修复和秘密发现。
- `短闭环高频`：玩家应在 30 秒内看到可互动内容，2 分钟内完成一个小闭环，10 分钟内拿到明确进展。
- `软失败可恢复`：失败可以损失未结算资源，但不能抹掉主线关键推进和长期学习成果。
- `做深数学，不做扩科`：后续迭代优先加深数学知识树和区域机制，而不是重新开一套别的学科体系。

## 首版体验骨架

首版单次游玩的理想流程应固定为：

1. 在主城选择本次出城的 `主目标`
2. 进入数学知识点区域后，快速遇到怪物、资源点、机关或分叉路线
3. 通过答题完成战斗、解谜、捷径解锁和资源回收
4. 击败区域精英或 Boss，拿到阶段性推进物
5. 回城结算资源、查看数学进展、升级建筑或研究工具
6. 明确知道下次应该去哪里、补什么、为什么值得再出城

首版允许的常见出城意图至少包含：

- `主线推进`：解锁新知识点区域、拿关键收集物、挑战 Boss
- `修错题`：带着明确的薄弱点回到对应区域复习和修复
- `采资源`：回收金币、材料和数学币，支撑主城成长
- `找秘密`：开启捷径、奖励宝箱、隐藏事件和收藏内容

## 跨模块硬约束

以下规则跨所有子文档统一生效：

- 每个知识点区域至少要有 `1 个地标`、`1 个安全点`、`1 个 Boss`、`1 个捷径/回程优化点`。
- 每个知识点都要同时存在 `学习进度` 和 `世界进度`，不能只在分数层面存在。
- 战斗中的答题结果必须直接映射到行动效果，不能变成空壳动画触发器。
- 主线机关以 `单题快闭环` 为主，可选奖励机关才允许稍微复杂。
- 错题不能只存档，必须能在后续探索、复习或主城中形成可控回流。
- 主城升级必须明确告诉玩家 `解锁了什么世界能力`、`改变了哪条探索路径`、`补强了哪个数学环节`。
- 首版必须支持 `5 分钟`、`10 分钟`、`20 分钟` 三种游玩时长，都能拿到可感知收益。

## 已确认的关键决策

### 主线体验

- 主线采用 `常驻世界自由探索`
- 世界由 `主城 + 数学大陆` 组成
- 数学大陆按 `知识点区域` 逐步开放
- 每个区域对应 `一个数学知识点`
- 区域开放关系由 `知识树前置图` 决定
- 遇怪后进入 `独立战斗界面`
- 战斗采用 `答题驱动 + 动作选择`
- 玩家是 `单人冒险`
- 每个知识点区域都有 `最终 Boss`
- 击败 Boss 后获得对应 `徽章` 并解锁区域传送

### 探索与回收

- 区域推进由 `知识点前置关系 + 区域探索完成度` 共同决定
- 主线关键收集物和 `徽章` 拿到即永久记录
- 野外会掉落 `材料`、`金币` 和 `数学币`
- 普通资源 `带回城才结算`
- 战败是 `软失败`
- 通过 `安全点/传送碑` 回城

### 主城成长

- 主城内包含 `建筑成长系统`
- 家园支持消耗资源 `建造和升级建筑`
- `金币` 是所有建筑共用的基础货币
- `数学币` 承接首版数学主线成长
- 建筑功能位已明确，数值和平衡后续再细化
- 不以自由摆放和重装饰为首版重点

### 知识树推进

- 数学大陆整体由 `数学知识树图` 组织
- `节点 = 一个知识点区域`
- `边 = 前置知识关系`
- 图允许分叉、汇合和多种结构，不预设固定拓扑
- 具体节点清单后续在知识树文档中补充
