# 学科探索答题游戏需求总览

这份文档是主文档，只负责说明产品总方向、模块边界和子文档目录。详细需求拆到子文档中，后续讨论和实现优先直接修改对应模块文档，而不是继续把所有内容堆回一份总稿。

## 当前产品方向

当前方向已经从“纯答题 + 城建”调整为：

- `主线核心`：常驻世界自由探索的学习 RPG
- `长期成长`：功能型城镇与学科成长体系
- `学习载体`：战斗、机关、收集、区域推进都通过题目驱动
- `轻社交`：排行榜、参观、异步协作事件

一句话定义：

> 玩家在学科主题的开放区域中探索、解谜、遇怪答题、带回资源升级城镇，并逐步解锁更深区域与更多策略玩法。

## 已确认的关键决策

### 主线体验

- 主线采用 `常驻世界自由探索`
- 世界按 `分区开放`
- 每个区域有 `主学科`
- 遇怪后进入 `轻回合制`
- 每回合 `先选行动，再答题判定`
- `答对成功，答错失败`
- 战斗以 `单怪为主，少量双怪`
- 玩家是 `单人冒险`

### 角色成长

- 采用 `主职业 + 副职业`
- 主职业决定核心战斗风格
- 副职业只提供 `1 个辅助技能 + 1 个被动`
- 区域决定题目主学科，职业不改题目学科
- 职业不限制进区，只影响战斗手感和技能结构

### 探索与回收

- 区域推进主要靠 `探索收集物`
- 主线关键收集物 `拿到即永久记录`
- 普通资源 `带回城才结算`
- 战败是 `软失败`
- 通过 `安全点/传送碑` 回城

### 城镇成长

- 城镇采用 `功能型城镇`
- 成长主要给 `新能力解锁`
- 首批成长优先解锁 `通用探索工具`
- 不以自由摆放和重装饰为首版重点

### 扩展模块

- 轻社交保留，但不喧宾夺主
- 当前首要验证对象仍然是 `探索答题 RPG 主线`
