# 数学大陆探索需求

## 模块目标

把原来的固定答题回合改造成更有代入感的数学冒险体验：

`常驻世界自由探索 -> 遭遇 -> 数学答题战斗/解谜 -> 资源回收 -> 回城成长`

玩家应该感觉自己是在数学大陆中冒险，而不是为了做题而被迫点下一轮。

## 体验承诺

这个模块要兑现的不是“地图里能放题”，而是下面这件事：

- 玩家在地图里 `看到目标`
- 玩家为靠近目标做 `路径和风险决策`
- 玩家在遭遇或机关里 `使用数学完成动作`
- 玩家带着明确收益 `回城结算`

如果玩家的主观感受还是“又进了一张题目地图”，说明这个模块设计失败。

## 出城目标结构

玩家每次出城都应至少带着一个主目标，并允许顺手完成一个副目标。

首版主目标类型：

- `推进主线`：解锁区域、拿关键收集物、挑战 Boss
- `修复薄弱点`：回到对应知识点区域，修错题、练稳定度
- `收集资源`：带回金币、材料、数学币，支撑主城升级
- `搜寻秘密`：开捷径、拿隐藏宝箱、完成收藏或支线事件

设计要求：

- 玩家出城前能明确看到当前推荐目标
- 同一区域应支持多种目标并存，避免“这次来只能打一件事”
- 副目标应自然嵌在主路线附近，不要求额外绕极远的路

## 世界结构

### 常驻世界

- 世界长期存在，不是一次性关卡
- 玩家在连通地图中自由移动
- 区域按知识点前置关系逐步开放

### 首版地图组成

`P0` 首版世界只包含两块核心地图：

- `主城`
- `数学大陆`

主城作为出发、回收和成长中枢。  
数学大陆承担首版全部探索内容。

### 数学大陆与知识树

数学大陆不是平铺关卡列表，而是 `知识树映射到世界后的探索图`。

规则：

- 每个区域对应 `一个数学知识点节点`
- 区域开放关系受 `05-knowledge-tree` 中的前置关系控制
- 玩家推进的核心不是单纯等级，而是知识点节点完成情况
- 区域开放顺序要服务数学学习结构，而不是只服务地图长度

## 知识点区域规范

每个知识点区域不只是“题库容器”，而是一段有节奏的可重复进入的冒险空间。

每个区域至少应具备：

- 当前知识点主题
- 区域入口和解锁条件
- 区域地标
- 区域 Boss
- 区域安全点
- 区域捷径
- 区域主机关类型
- 区域主战斗风格
- 区域阶段奖励

## 区域模板

首版建议每个区域至少包含以下功能段：

- `入口段`：快速说明这个区域的数学主题、风险和可去方向
- `教学段`：用低压怪物或简单机关让玩家理解本区主要数学动作
- `变化段`：出现路线分叉、资源诱因、奖励机关或主题变体
- `风险段`：怪物压力上升，机关和路线选择更明显，逼迫玩家判断回城或继续
- `Boss 段`：该知识点的综合检验区

每个区域至少应具备：

- `1 个强地标`
- `1 个安全点`
- `1 条捷径`
- `1 个可选奖励区`
- `1 个回城前诱因`

## 数学区域表达

数学大陆里的区域主题必须让玩家感受到“数学在世界里被使用”，而不是只看到一层题面。

区域表达至少围绕以下四类数学动作展开：

- `数量判断`
- `规律识别`
- `空间理解`
- `关系推演`

表现方式可以体现在：

- 怪物行为和弱点
- 机关结构和解锁方式
- 路径提示和安全路线
- 资源选择和收益判断

## 探索节奏模板

首版自由探索必须显式按节奏设计，避免地图空走。

建议节奏如下：

- `30 秒内`：玩家应看到怪物、资源点、门锁、机关、分叉路或地标中的至少一个
- `2 分钟内`：玩家应完成一次小闭环，例如打一场战斗、开一个机关、拿一个宝箱或发现一条捷径
- `10 分钟内`：玩家应获得一次明确进展，例如推进一段路线、修复一组错题、拿到关键收集物、解锁安全点或击败精英

地图设计约束：

- 长走廊必须用可见目标、风险预告或收集激励打断
- 奖励内容不能全部放在线性主路尽头
- 每个区域都应有至少一条“安全但收益低”和一条“风险高但收益好”的选择路线

## 野外内容构成

自由探索不能只有怪物。首版野外应至少包含：

- 怪物遭遇
- 资源点
- 宝箱
- 单题机关
- 路线封印 / 门锁 / 石碑
- 主线收集物

进一步要求：

- `主线必经内容` 只占区域内容的一部分，必须保留探索型支线和可选奖励
- `资源点` 不应只是点击领取，应尽量绑定轻量观察或短交互
- `宝箱` 应有类型区分，例如资源箱、捷径箱、知识收藏箱
- `主线收集物` 必须和知识点主题有关，不能只是抽象钥匙
- `重复进入` 区域时，玩家应因为捷径、Boss 已击败状态、回流题目标不同而产生新路线选择

## 战斗结构

### 基本规则

- 单人冒险
- 野外接触怪物后进入 `独立战斗界面`
- 战斗核心是 `答题驱动`
- 首版只保留少量动作类型，重点验证数学题和动作绑定是否自然

### 玩家动作层

战斗不能只是“出题后自动结算”，玩家必须在答题前做动作选择。

首版基础动作建议为：

- `普攻`：稳定输出，对应短平快题
- `技能`：消耗资源释放，对应难度更高或更综合的题
- `防御`：降低本回合受伤，对应更偏识别和稳答的题
- `逃跑`：尝试脱战，对应时长更短的快速判定题
- `道具`：首版可简化为少量恢复或辅助，不强制每次都答题

标准回合流程建议为：

1. 玩家选择动作
2. 系统根据动作、怪物类型和当前知识点派发题目
3. 按 `完美 / 正确 / 失误` 三档结算效果
4. 敌方执行回合
5. 更新战斗资源、状态和是否进入下一轮

动作和答题效果的绑定原则：

- `完美`：更高伤害、更强防御、更高逃跑成功率或额外收益
- `正确`：动作正常生效
- `失误`：动作弱化或失败，并承受敌方压力

### 战斗资源

首版建议引入一种轻量公共战斗资源，例如 `专注值`。

其作用：

- 通过连续正确作答积累
- 作为释放技能、反制强敌或保底恢复的资源
- 让玩家除了“答对/答错”之外，还能感受到战斗内节奏变化

### 怪物类型

为了让题目在战斗中呈现出策略差异，首版怪物至少区分以下倾向：

- `冲锋型`：高攻击，逼玩家重视防御和快速击破
- `护盾型`：容错高，逼玩家用技能或连续正确作答破防
- `干扰型`：造成状态压力，逼玩家权衡稳答还是保资源
- `蓄力型`：会提前预告强招，逼玩家做节奏判断

怪物的差异应体现在 `玩家为何选择不同动作`，而不是只换一套皮肤和数值。

### 怪物难度

- 同一区域会放置不同难度层级的怪物
- 怪物难度对应该知识点下的题目难度层级
- 玩家在同一知识点区域内，也应感受到从基础到综合的难度爬坡

首版难度爬坡建议：

- `普通怪`：单点考察，建立熟练度
- `精英怪`：开始混入变体、干扰或资源压力
- `Boss 前哨怪`：提示 Boss 将考哪些能力
- `Boss`：对这个知识点进行综合检验

### Boss

每个知识点区域都有一个 `最终 Boss`。

它的作用不是单题卡门，而是作为这个知识点的综合考察。

如果玩家已经真正掌握该知识点，就应当能自然击败 Boss。

击败 Boss 后，玩家会获得：

- 该知识点对应的 `徽章`
- 该区域的长期完成标记
- 后续直接传送到该区域的能力

Boss 设计要求：

- 不是“一题定生死”，而是一次短程综合战
- 至少覆盖该知识点下的两到三类核心能力
- 玩家在挑战前应能从主城或区域提示中看到 `Boss 就绪信号`
- 首次失败后应能知道自己差在 `正确率`、`稳定度` 还是 `某类题型空缺`

## 解谜结构

首版谜题采用 `题目驱动解谜`，但限制为 `单题机关为主`。

标准流程：

1. 玩家观察环境线索
2. 接触机关 / 石碑 / 封印点
3. 出一题
4. 答对后开门、放桥、解锁宝箱、解除阻挡或推进区域

这样能在首版低成本下建立“探索不是刷怪，而是边看边解边推进”的感觉。

### 数学机关类型

首版机关建议分为三类：

- `主线机关`：必须通过，单题快闭环，失败成本低
- `奖励机关`：可选，通过后给宝箱、收藏、资源或捷径
- `捷径机关`：首次需要绕路或解锁，成功后优化后续跑图

数学表达可以落在：

- 序列和规律门
- 坐标与路径机关
- 配比与平衡装置
- 图形和空间旋转机关
- 关系推导型符文机关

### 机关设计约束

- 主线机关优先考 `理解与应用`，不要考超长作答
- 同一条主线连续出现多个强制机关时，必须用战斗、收集或对话打断
- 奖励机关可以略难，但失败后应提供提示、绕路或稍后回来再解的空间
- 机关前应先有环境提示，不要让题目脱离场景单独存在

## 推进与失败

### 区域推进

推进主要靠 `知识点前置关系 + 区域探索完成度`。

例如：

- 前置知识点完成
- 钥匙
- 徽记
- 古文碎片
- 区域证明物

### 关键规则

- 主线关键收集物和徽章拿到即永久记录
- 战败不会抹掉主线推进
- 安全点、捷径、已解机关等世界状态应尽量永久保留
- 玩家重返区域时，应优先感受到“熟路越来越顺”，而不是“又得从头跑”

### 失败代价

采用 `软失败`：

- 战败后回城
- 损失部分未结算普通资源
- 保留关键推进物和长期进度

首版挫败保护要求：

- 连续失败后，应给出更明确的推荐去向，例如回城复习、先打其他支线、先升级工具
- 不允许因为一次失败把玩家卡回很久以前的进度
- 回城后应能看懂这次失败主要来自哪类问题

## 回收与回城

### 普通资源

- 野外获取后先存在背包中
- 包括材料、金币和数学币
- 只有回城后才正式结算入库

### 回城方式

采用 `安全点 / 传送碑回城`

- 不是随时一键无损撤退
- 保留“再往前一点”的张力
- 也避免硬核到劝退

安全点设计要求：

- 首次进入区域后的短时间内应能看到第一个安全点
- 安全点之间的距离要足以制造风险，但不能长到让玩家不敢继续深入
- 安全点应兼具 `回城`、`进度暂停`、`轻提示` 三种作用

### 区域传送

击败区域 Boss 并获得对应徽章后，玩家后续可以直接传送到该区域。

这样可以避免重复跑图，同时保留首次探索和首次攻克的价值。

## 单次游玩时长适配

首版每个区域都应能兼容不同 session：

- `5 分钟`：刷一段短线、打一组怪、解一个机关、顺路回城
- `10 分钟`：完成一个明确小目标，例如修掉一组错题或打通一条捷径
- `20 分钟`：做一次深探或 Boss 挑战

地图和系统要共同保证：

- 碎片时间玩家不会因为路途过长而不敢进入
- 长 session 玩家也有足够的深度内容和风险回报

## 首版约束

- 不做多人组队冒险
- 不做复杂大型区域谜题
- 不做完整开放大地图一次性全开
- 不把战斗细则写到难以调整，但必须先明确动作层和资源层
- 不让任何区域只剩单一路线推进
- 不让地图内容只服务一次性首通，不考虑复访体验

## 首版验证问题

- 玩家是否比固定回合更愿意主动探索连通地图
- 知识点区域结构是否能提升数学主题感
- Boss + 徽章 + 传送是否能形成清晰阶段目标
- 战斗答题是否足够自然
- 解谜答题是否真的增强探索感而非打断感
- 玩家是否能清楚说出“这次出城我的目标是什么”
- 玩家重返旧区域时是否仍然有意义而不是单纯重复
