# 主线探索 RPG 需求

## 模块目标

把原来的固定答题回合改造成更有代入感的主线体验：

`常驻世界自由探索 -> 遭遇 -> 答题战斗/解谜 -> 资源回收 -> 回城成长`

玩家应该感觉自己是在冒险中学习，而不是为了做题而被迫点下一轮。

## 世界结构

### 常驻世界

- 世界长期存在，不是一次性关卡
- 玩家在连通地图中自由移动
- 区域按知识点前置关系逐步开放

### 首版地图组成

首版世界只包含三块核心地图：

- `主城`
- `数学大陆`
- `英语大陆`

主城作为出发、回收和成长中枢。  
两个大陆承担首版学科探索内容。

### 分区开放

大陆不会整块一次性全部开放，而是继续拆成多个区域。

每个区域对应 `一个知识点`，并且这些区域不是平铺列表，而是组成一张知识点推进图。

每个知识点节点应至少具备：

- 前置节点
- 所属大陆
- 当前知识点主题
- 对应区域入口
- 对应区域 Boss

每个区域具备：

- 主学科主题
- 单一知识点主题
- 自己的怪物风格
- 自己的机关和资源主题
- 自己的探索收集物

## 区域与学科

采用 `区域主学科` 规则。

- 玩家进入什么区域，主要接触该区域对应学科题
- 同一大陆下的区域按知识点拆分
- 区域决定世界感和学习主题

这个规则的目标是让“世界内容”和“学习内容”天然绑定。

## 知识点推进图

知识点之间不是线性清单，而是有前置条件的节点图。

规则：

- 完成前置知识点后，后续知识点区域才会开启
- 玩家推进的核心不是单纯等级，而是知识点节点完成情况
- 区域开放顺序要服务学习结构，而不是只服务地图长度

这能让世界推进和知识点学习进度保持一致。

## 野外内容构成

自由探索不能只有怪物。首版野外应至少包含：

- 怪物遭遇
- 资源点
- 宝箱
- 单题机关
- 路线封印 / 门锁 / 石碑
- 主线收集物

## 战斗结构

### 基本规则

- 单人冒险
- 野外接触怪物后进入 `独立战斗界面`
- 战斗核心是 `答题驱动`
- 详细战斗规则后续再细化

### 怪物难度

- 同一区域会放置不同难度层级的怪物
- 怪物难度对应该知识点下的题目难度层级
- 玩家在同一知识点区域内，也应感受到从基础到综合的难度爬坡

### Boss

每个知识点区域都有一个 `最终 Boss`。

它的作用不是单题卡门，而是作为这个知识点的综合考察。

如果玩家已经真正掌握该知识点，就应当能自然击败 Boss。

击败 Boss 后，玩家会获得：

- 该知识点对应的 `徽章`
- 该区域的长期完成标记
- 后续直接传送到该区域的能力

## 解谜结构

首版谜题采用 `题目驱动解谜`，但限制为 `单题机关为主`。

标准流程：

1. 玩家观察环境线索
2. 接触机关 / 石碑 / 封印点
3. 出一题
4. 答对后开门、放桥、解锁宝箱、解除阻挡或推进区域

这样能在首版低成本下建立“探索不是刷怪，而是边看边解边推进”的感觉。

## 推进与失败

### 区域推进

推进主要靠 `知识点前置关系 + 区域探索完成度`。

例如：

- 前置知识点完成
- 钥匙
- 徽记
- 古文碎片
- 区域证明物

### 关键规则

- 主线关键收集物和徽章拿到即永久记录
- 战败不会抹掉主线推进

### 失败代价

采用 `软失败`：

- 战败后回城
- 损失部分未结算普通资源
- 保留关键推进物和长期进度

## 回收与回城

### 普通资源

- 野外获取后先存在背包中
- 包括材料、金币和大陆专属货币
- 只有回城后才正式结算入库

### 回城方式

采用 `安全点 / 传送碑回城`

- 不是随时一键无损撤退
- 保留“再往前一点”的张力
- 也避免硬核到劝退

### 区域传送

击败区域 Boss 并获得对应徽章后，玩家后续可以直接传送到该区域。

这样可以避免重复跑图，同时保留首次探索和首次攻克的价值。

## 首版约束

- 不做多人组队冒险
- 不做复杂大型区域谜题
- 不做完整开放大地图一次性全开
- 不把战斗细则在首版需求文档里提前写死

## 首版验证问题

- 玩家是否比固定回合更愿意主动探索连通地图
- 知识点区域结构是否能提升学习主题感
- Boss + 徽章 + 传送是否能形成清晰阶段目标
- 战斗答题是否足够自然
- 解谜答题是否真的增强探索感而非打断感
