# 主线探索 RPG 需求

## 模块目标

把原来的固定答题回合改造成更有代入感的主线体验：

`常驻世界自由探索 -> 遭遇 -> 答题战斗/解谜 -> 资源回收 -> 回城成长`

玩家应该感觉自己是在冒险中学习，而不是为了做题而被迫点下一轮。

## 世界结构

### 常驻世界

- 世界长期存在，不是一次性关卡
- 玩家在地图中自由移动
- 区域按进度逐步开放

### 分区开放

首版采用多个主题区域，而不是完整一次性大地图。

每个区域具备：

- 主学科主题
- 自己的怪物风格
- 自己的机关和资源主题
- 自己的探索收集物

## 区域与学科

采用 `区域主学科` 规则。

- 玩家进入什么区域，主要接触该区域对应学科题
- 区域决定世界感和学习主题
- 职业不改写区域主学科，只改写战斗手感

这个规则的目标是让“世界内容”和“学习内容”天然绑定。

## 野外内容构成

自由探索不能只有怪物。首版野外应至少包含：

- 怪物遭遇
- 资源点
- 宝箱
- 单题机关
- 路线封印 / 门锁 / 石碑
- 主线收集物

## 战斗结构

### 基本规则

- 单人冒险
- 轻回合制
- 单怪为主，少量双怪

### 回合流程

1. 玩家先选行动
2. 系统出当前区域主学科题
3. 答对则行动成功
4. 答错则行动失败
5. 怪物执行其回合

### 可选行动

首版建议保留以下行动：

- 普攻
- 主职业技能
- 副职业辅助技能
- 防御
- 道具
- 逃跑

## 职业结构

### 主职业

- 决定核心战斗风格
- 决定主技能线和主要成长方向

### 副职业

- 只提供 `1 个辅助技能 + 1 个被动`
- 不作为第二套完整出手体系

### 职业自由度

- 职业不限制进入哪些区域
- 区域也不强制要求特定职业
- 差异主要体现在战斗体验和技能搭配

## 解谜结构

首版谜题采用 `题目驱动解谜`，但限制为 `单题机关为主`。

标准流程：

1. 玩家观察环境线索
2. 接触机关 / 石碑 / 封印点
3. 出一题
4. 答对后开门、放桥、解锁宝箱、解除阻挡或推进区域

这样能在首版低成本下建立“探索不是刷怪，而是边看边解边推进”的感觉。

## 推进与失败

### 区域推进

推进主要靠 `探索收集物`，而不是强制 Boss 门槛。

例如：

- 钥匙
- 徽记
- 古文碎片
- 区域证明物

### 关键规则

- 主线关键收集物拿到即永久记录
- 战败不会抹掉主线推进

### 失败代价

采用 `软失败`：

- 战败后回城
- 损失部分未结算普通资源
- 保留关键推进物和长期进度

## 回收与回城

### 普通资源

- 野外获取后先存在背包中
- 只有回城后才正式结算入库

### 回城方式

采用 `安全点 / 传送碑回城`

- 不是随时一键无损撤退
- 保留“再往前一点”的张力
- 也避免硬核到劝退

## 首版约束

- 不做多人组队冒险
- 不做复杂大型区域谜题
- 不做完整开放大地图一次性全开
- 不做大规模怪群战
- 不做高操作即时战斗

## 首版验证问题

- 玩家是否比固定回合更愿意主动探索
- 区域主学科是否能提升学习主题感
- 战斗答题是否足够自然
- 解谜答题是否真的增强探索感而非打断感
