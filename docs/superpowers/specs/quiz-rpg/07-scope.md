# 范围、风险与验证

## 当前版本切分

## Phase 1：探索答题 RPG 核心验证

这是当前最优先阶段，必须先证明以下主链路成立：

`自由探索 -> 遇怪答题 -> 解谜推进 -> 带资源回城 -> 解锁新能力 -> 继续探索`

Phase 1 必须覆盖：

- 常驻世界自由探索
- 分区开放
- 区域主学科
- 轻回合制单人战斗
- 题目驱动机关
- 功能型城镇
- 通用探索工具
- 学习进步反馈

## Phase 2：轻社交增强

在主线成立后，再叠加：

- 排行榜
- 参观
- 异步协作事件

目的是强化陪伴感和回流，而不是改变主玩法。

## Phase 3：异步塔防竞技层

等主线与轻社交都稳定后，再接入：

- 异步攻防
- 段位与赛季
- 技能触发题的竞技版本

## 当前最大风险

### 风险 1：自由探索内容空心化

问题：

- 如果野外只有怪和题，自由探索会很快变成换皮刷题。

缓解：

- 保证资源点、机关、宝箱、封印、主线收集物并存。

### 风险 2：答题打断 RPG 节奏

问题：

- 如果战斗和机关频率太高、题目太长，探索会被切碎。

缓解：

- 首版控制题量和交互时长。
- 战斗保持轻回合制，机关保持单题结构。

### 风险 3：城镇成长没有爽点

问题：

- 如果回城只是存资源，成长价值会很弱。

缓解：

- 让成长优先表现为探索能力解锁。

### 风险 4：学习反馈仍然不可见

问题：

- 如果只是换了壳，玩家还是不会觉得自己在变强。

缓解：

- 必须做知识点进步、错题修复和阶段表现反馈。

### 风险 5：范围膨胀

问题：

- 同时做开放探索、复杂城镇、深社交、塔防竞技，首版会失控。

缓解：

- 先压住 Phase 1。
- 轻社交和塔防按阶段后置。

## 当前不做

- 实时同步 PvP
- 多人组队冒险
- 自由城市摆放
- 大型区域谜题
- 复杂加工链
- 重公会系统
- 首版深挖塔防竞技

## 核心验证指标

### 主线体验

- 单次探索时长
- 回城结算频率
- 区域推进完成率
- 安全点使用频率

### 学习效果

- 知识点表现变化
- 错题修复率
- 区域主学科完成情况

### 留存

- 次日回流率
- 7 日回流率
- 每日主动探索次数
- 回城后再次出城比例

## 成功标准

如果 Phase 1 要证明方向成立，至少要看到：

- 玩家愿意主动出城，而不是只被任务推着走
- 玩家能说出某个区域在“学什么”
- 玩家能感知自己在某些知识点上变强
- 玩家回城后愿意继续升级并再次出发
- 自由探索的吸引力明显高于固定 5 题回合
