# 产品总览与路线图

## 产品定位

这是一款学习型探索 RPG。题目不再只是回合奖励入口，而是直接嵌入探索、战斗、解谜和区域推进。

产品价值由四层组成：

- `学习价值`：玩家能明确感知知识点进步
- `探索价值`：世界、区域、机关和收集驱动持续前进
- `成长价值`：回城升级设施与能力，打开新区域和新玩法
- `社交价值`：轻社交增强陪伴和回流

## 产品目标

- 让用户因为探索和遭遇感愿意主动做题
- 让玩家感知“题目是行动的一部分”，而不是外挂结算
- 让回城成长形成长期目标，而不是一次性清关
- 让世界推进和学习推进相互强化

## 非目标

- 不做实时同步 PvP
- 不做资源掠夺
- 不做重公会和深社交关系链
- 不做完全开放的自由城建
- 不做首版即包含多套高复杂度玩法并行深挖

## 版本策略

### 当前优先级

`P0` 是把探索答题 RPG 主线做顺。  
也就是先验证：

- 自由探索是否比固定 5 题回合更有吸引力
- 区域主学科是否能让学习结构更自然
- 战斗与机关里的答题是否足够有趣而不劝退
- 回城成长是否能驱动持续探索

### 后续模块

`P1` 继续加轻社交，强化陪伴和回流。

## 核心产品结构

### 主循环

`出城探索 -> 遇怪/解谜/收集 -> 带回资源 -> 回城解锁能力 -> 进入更深区域`

### 支撑循环

- `学习循环`：知识点进步、错题修复、学科掌握反馈
- `成长循环`：设施升级、探索工具、职业能力和区域权限
- `轻社交循环`：排行榜、参观、协作事件

## 模块边界

### 探索 RPG

负责主体验、区域推进、战斗、解谜、风险回收。

### 学习系统

负责题目组织、知识点标签、学习反馈、重复控制。

### 城镇系统

负责资源回收、能力解锁、设施成长、区域推进支撑。

### 轻社交系统

负责陪伴感、参照感、协作回流。

## 当前最大约束

- 不要再让文档默认“城建是主体验”
- 不要让世界探索变成仅仅“换地图刷题”
- 不要让自由探索首版被复杂解谜和复杂城镇拖死
