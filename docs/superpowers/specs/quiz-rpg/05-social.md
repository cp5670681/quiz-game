# 轻社交系统需求

## 模块定位

轻社交不是主体验，但它要承担三件事：

- 提供陪伴感
- 提供参照感
- 提供回流理由

首版不做强社交链，不做复杂协作组织。

## 核心内容

### 排行榜

首版建议保留：

- 总进度榜
- 学科成长榜
- 区域探索榜

榜单应避免直接变成纯在线时长榜。

### 参观

玩家可以查看其他玩家的：

- 城镇成长状态
- 主职业 / 副职业方向
- 区域推进阶段

价值在于：

- 激发模仿
- 提供追赶目标
- 形成世界中“别人也在冒险”的感觉

### 异步协作事件

首版建议保留一个低复杂度模板：

- 大家各自做自己的探索 / 战斗 / 学习
- 系统汇总为全服或小组共同进度
- 阶段达成后发放通用奖励

## 与主线关系

轻社交要服务主线，不应要求玩家中断主线去做单独社交劳动。

正确方式是：

- 你正常探索时就会贡献事件进度
- 你正常成长时就会出现在参照体系里
- 你正常学习时就能获得社交反馈

## 边界

- 不做实时聊天依赖
- 不做公会组织
- 不做强制组队
- 不做资源互抢
- 不做需要多人在线配合的核心内容

## 首版验证问题

- 玩家是否会主动看别人进度
- 协作事件是否能成为额外回流理由
- 轻社交是否增强世界活跃感而不增加负担
